Introduction

The account types on GitHub

GitHub offers three types of accounts.

  • Personal account:
    Personal accounts are designed for individual users who are managing personal projects or collaborating with others.

  • Organization account:
    Organization accounts are designed to manage team projects with advanced management features. They have separate pricing plans.

    • GitHub Free:
      The GitHub Free plan is designed for small teams. This plan offers basic features like unlimited public and private repositories, 2,000 minutes/month of GitHub Actions, and 500 MB of GitHub Packages storage.

    • GitHub Team:
      The GitHub Team plan is designed for teams that need more advanced collaboration and management features, such as multiple reviewers in pull requests, draft pull requests, and required reviewers. This plan also provides 3,000 minutes/month of GitHub Actions and 2 GB of GitHub Packages storage.

    • GitHub Enterprise:
      The GitHub Enterprise plan is designed for large businesses. It offers advanced security, compliance, and management features.

  • Enterprise account:
    Enterprise accounts are accounts that own organizations. Unlike organization and personal accounts, enterprise accounts cannot create repositories.

In this article, I'd like to delve into Organization accounts — how to create one and what settings can be configured.

Create a new organization account

First, we need to create an organization account. The following are the steps to create a new organization account:

  1. Go to https://github.com/organizations/plan.
  2. Select a plan: Free, Team, or Enterprise.
  3. On the setup page, fill in the Organization name, Contact email, and the option under "This organization belongs to."

Configure organization default settings

In the organization, we can set default configurations for the repositories that will be created in the organization. This can be done by going to Setting > Member privileges.

Create a .github repository

As with a personal account, the .github repository is a special repository that stores organization-wide configuration and template files. In organizations, the defaults will be applied to any repository that doesn’t have its own version of these files.

Here are some of the files you can store in a .github repository:

  • CODE_OF_CONDUCT.md:
    Outlines the expected behavior for contributors and participants in a project.

  • CONTRIBUTING.md:
    Explains how people can contribute to the organization.

  • SECURITY.md:
    Provides instructions for reporting security vulnerabilities.

  • SUPPORT.md:
    Offers instructions for getting help.

  • FUNDING.yml:
    A configuration file used by GitHub to display the Sponsor button on repository pages.

  • ISSUE_TEMPLATE.md or ISSUE_TEMPLATE/*.md:
    Provides default issue templates for all repositories in the organization.

  • PULL_REQUEST_TEMPLATE.md or PULL_REQUEST_TEMPLATE/*.md:
    Provides default pull request templates for all repositories in the organization.

  • workflows/*.yml:
    Defines reusable workflows that can be used by repositories in the organization.

  • workflow-templates/*.yml:
    Defines workflow templates that can be used by repositories in the organization.

Create a team

In the organization, teams let you group members, manage access, and collaborate efficiently.

For example, you can manage repository access by assigning team-level permissions to repositories (read, write, maintain, admin).

For efficient collaboration, you can use @<org name>/<team name> to mention all members of a team in issues, pull requests, discussions, and comments. You can require code reviews from a specific team. Teams can also have private discussion boards.

Teams can be nested.
